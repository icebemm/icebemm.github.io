新增或更新

on duplicate key update: 不存在则新增insert，存在则更新updatge

    insert into table1 (key1,key2,key3) values (value1,value2,value3) on duplicate key update key3=value4
table1表，其中key1为主键或唯一键
1.当表中不存在(value1,value2,value3)值的之后，执行insert新增操作，值为(value1,value2,value3)
2.当主键或唯一键（key1）value1值在表中数据存在时（不管value2和value3数据是什么样的），
    执行后边的update更新操作，更新key3=value4，此时表中其余值value2不变

注意3bug

1. 当表中存在多个唯一键时，是不安全的
    当mysql执行INSERT ON DUPLICATE KEY的 INSERT时，存储引擎会检查插入的行是否会产生重复键错误。如果是的话，它会将现有的
    行返回给mysql，mysql会更新它并将其发送回存储引擎。当表具有多个唯一或主键时，此语句对存储引擎检查密钥的顺序非常敏感。根据这个顺序，
    存储引擎可以确定不同的行数据给到mysql，因此mysql可以更新不同的行。存储引擎检查key的顺序不是确定性的。例如，InnoDB按照索引添加到
    表的顺序检查键。

2. 在有可能有并发事务执行时候，会导致death lock死锁
    insert ... on duplicate key 在执行时，innodb引擎会先判断插入的行是否产生重复key错误，
    如果存在，在对该现有的行加上S（共享锁）锁，如果返回该行数据给mysql,然后mysql执行完duplicate后的update操作，
    然后对该记录加上X（排他锁），最后进行update写入。

3. ON DUPLICATE KEY UPDATE不能写where条件
    如果行作为新记录被插入，则受影响行的值为1；如果原有的记录被更新，则受影响行的值为2，如果更新的数据和已有的数据一模一样，则受影响的行数是0

多个表数据合并输出

业务场景：

要将不同的表数据或查询统计出的数据合并一快数据输出（此时是有多个id需要进行分组）；如统计状态为3的数据进行id分组，同时又要统计出每个id的所有状态个数。
代码如下：
select t1.id,t1.name,t1.age,t1.num1,t2.sex,t2.num2 from 
	(select a.id,a.name,a.age,count(id) as num1 from a where status=3 group by a.id) t1
	left join
	(select b.id,b.sex,count(id) as num2 from b group by b.id) t2
	on t1.id = t2.id
